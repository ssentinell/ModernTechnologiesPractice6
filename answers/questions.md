# Контрольные вопросы - Практика 6

## Вопрос 1: Что такое Maven?

**Вопрос:** Что такое Maven и зачем он нужен?

**Ваш ответ:**  
Maven — это инструмент сборки и управления зависимостями для Java-проектов. Он помогает стандартизировать структуру проекта, автоматически подключать библиотеки, запускать типовые стадии сборки (`compile`, `test`, `package`, `install`) и хранить конфигурацию в `pom.xml`.

---

## Вопрос 2: Прямые vs Транзитивные зависимости

**Вопрос:** Чем отличается прямая зависимость от транзитивной?

**Ваш ответ:**  
Прямая зависимость — это библиотека, которую разработчик сам указал в `pom.xml`.  
Транзитивная зависимость — это библиотека, которая нужна прямой зависимости и автоматически подтягивается Maven.

---

## Вопрос 3: PreparedStatement

**Вопрос:** Почему PreparedStatement безопаснее Statement?

**Ваш ответ:**  
`PreparedStatement` безопаснее, потому что использует параметры (`?`) вместо склейки SQL-строк. Это защищает от SQL Injection и делает код чище. Кроме того, один и тот же запрос можно переиспользовать с разными параметрами.

---

## Вопрос 4: ORM

**Вопрос:** Что такое ORM и какую проблему он решает?

**Ваш ответ:**  
ORM (Object-Relational Mapping) — это подход, при котором Java-объекты связываются с таблицами базы данных. ORM решает проблему несоответствия между объектной моделью приложения и реляционной моделью БД: разработчик работает с объектами, а не вручную преобразует строки таблиц в классы.

---

## Вопрос 5: HQL vs SQL

**Вопрос:** Чем HQL отличается от SQL?

**Ваш ответ:**  
SQL работает с таблицами и столбцами базы данных.  
HQL работает с Java-сущностями и их полями.  
То есть SQL ближе к структуре БД, а HQL — к объектной модели приложения.

## Дополнительные вопросы из задания

### 1. Что такое GAV-координаты в Maven? Для чего они используются?
GAV = `groupId`, `artifactId`, `version`.  
Эти три поля однозначно определяют библиотеку в Maven-репозитории и позволяют Maven скачать нужную зависимость.

### 2. В чём разница между `<scope>compile</scope>` и `<scope>test</scope>`?
- `compile` — зависимость нужна на этапе компиляции, тестов и выполнения
- `test` — зависимость нужна только для тестов, в итоговый runtime она не входит

### 3. Что такое транзитивные зависимости? Может ли это стать проблемой?
Это зависимости зависимостей. Да, иногда это становится проблемой из-за конфликтов версий, дубликатов библиотек и неожиданного роста дерева зависимостей.

### 4. Чем Gradle отличается от Maven? Назовите 2-3 преимущества каждого.
**Maven:**
- строгая и понятная структура
- удобен для типовых проектов
- проще читать `pom.xml`

**Gradle:**
- более гибкий
- быстрее и удобнее для сложных кастомных сборок
- build-скрипты обычно короче

### 5. Что такое JDBC Driver? Почему для разных СУБД нужны разные драйверы?
JDBC Driver — это библиотека, которая умеет “переводить” JDBC-вызовы Java в конкретный протокол работы определённой СУБД.  
У разных СУБД разные протоколы, диалекты SQL и особенности подключения, поэтому драйверы разные.

### 6. Чем PreparedStatement отличается от Statement? Зачем нужен PreparedStatement?
`Statement` обычно использует готовую строку SQL целиком.  
`PreparedStatement` сначала подготавливает шаблон SQL, а потом получает параметры. Он нужен для безопасности, удобства и повторного использования запросов.

### 7. Что такое SQL Injection? Как PreparedStatement защищает от неё?
SQL Injection — это атака, при которой пользовательский ввод внедряется в SQL и меняет его смысл.  
`PreparedStatement` отделяет структуру запроса от данных, поэтому пользовательский ввод не исполняется как SQL-код.

### 8. Что такое транзакция? Что означают свойства ACID?
Транзакция — это логически единый набор операций, который должен выполниться целиком или не выполниться вообще.

ACID:
- **Atomicity** — атомарность
- **Consistency** — согласованность
- **Isolation** — изоляция
- **Durability** — долговечность

### 9. Что такое ORM? Какие преимущества и недостатки по сравнению с чистым JDBC?
**Преимущества ORM:**
- меньше шаблонного кода
- работа через объекты
- проще CRUD и маппинг

**Недостатки ORM:**
- меньше контроля над SQL
- сложнее отладка производительности
- есть накладные расходы по абстракции

### 10. Что такое `@Entity` и `@Table` в Hibernate? Что происходит если они отсутствуют?
- `@Entity` помечает класс как сущность Hibernate/JPA
- `@Table` указывает имя таблицы

Если убрать `@Entity`, Hibernate не будет считать класс сущностью.  
Если убрать `@Table`, Hibernate обычно использует имя таблицы по имени класса по умолчанию.

### 11. Чем HQL отличается от SQL? Чем HQL отличается от Criteria API?
- HQL — строковый язык запросов Hibernate, ближе к SQL, но работает с сущностями
- SQL — работает напрямую с таблицами БД
- Criteria API — это программный, типизированный способ строить запросы через Java API без строк

### 12. Что означает `hbm2ddl.auto = create` в конфигурации Hibernate?
Это значит, что при запуске Hibernate создаёт схему базы заново. Обычно старая структура удаляется и создаётся новая. Это удобно для учебных задач, но опасно для production, потому что можно потерять данные.
