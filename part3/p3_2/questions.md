### Ответ 1: JDBC vs Hibernate

**JDBC (Java Database Connectivity):**
- Низкоуровневый API для работы с БД
- Прямая работа с SQL запросами
- Ручное управление соединениями
- Ручное маппинг ResultSet → объекты
- Больше boilerplate кода

**Hibernate (ORM):**
- ORM (Object-Relational Mapping) фреймворк
- Работа с объектами, а не с SQL
- Автоматическое управление сессиями
- Автоматическое маппинг объектов ↔ таблицы
- Меньше кода, выше абстракция
- HQL вместо SQL (хотя SQL тоже можно)

### Ответ 2: HQL vs SQL

**HQL (Hibernate Query Language):**
- Объектно-ориентированный язык запросов
- Работает с классами и полями, не с таблицами и столбцами
- `FROM Movie WHERE genre = 'sci-fi'` - Movie это класс, genre это поле
- Переносим между разными БД (Hibernate генерирует нужный SQL)
- Поддерживает наследование, полиморфизм

**SQL:**
- Структурированный язык запросов
- Работает с таблицами и столбцами
- `SELECT * FROM movies WHERE genre = 'sci-fi'`
- Зависит от диалекта БД

### Ответ 3: Аннотации Hibernate

**Основные аннотации на сущности Movie:**

1. **@Entity** - помечает класс как JPA сущность
2. **@Table(name = "movies")** - указывает имя таблицы в БД
3. **@Id** - помечает поле как первичный ключ
4. **@GeneratedValue(strategy = GenerationType.IDENTITY)** - автогенерация ID
5. **@Column(name = "title", nullable = false)** - настройка столбца:
   - `name` - имя столбца
   - `nullable` - может ли быть NULL

**Дополнительные распространенные аннотации:**
- **@OneToMany**, **@ManyToOne** - связи между сущностями
- **@Temporal** - для полей Date/Calendar
- **@Enumerated** - для enum полей
- **@Transient** - поле не сохраняется в БД
