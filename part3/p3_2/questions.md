# Вопросы по Hibernate - Часть 3

## Вопрос 1: JDBC vs Hibernate

**Вопрос:** В чем разница между JDBC и Hibernate?

**Ваш ответ:**  
JDBC — это низкоуровневый API для прямой работы с базой данных через SQL-запросы.  
Hibernate — это ORM-фреймворк, который позволяет работать с данными через Java-объекты и сам связывает их с таблицами БД.

Если кратко:
- в JDBC разработчик сам пишет SQL, сам управляет `Connection`, `PreparedStatement`, `ResultSet`
- в Hibernate разработчик работает с сущностями, а SQL часто генерируется автоматически

## Вопрос 2: HQL vs SQL

**Вопрос:** Что такое HQL и чем он отличается от SQL?

**Ваш ответ:**  
HQL (Hibernate Query Language) — это язык запросов Hibernate, который работает не с таблицами и колонками, а с Java-классами и их полями.

Пример:
- SQL: `SELECT * FROM movies WHERE genre = 'sci-fi'`
- HQL: `FROM Movie WHERE genre = :genre`

Разница:
- SQL работает на уровне таблиц БД
- HQL работает на уровне сущностей Java
- HQL более удобен в ORM-подходе и переносим между СУБД через Hibernate

## Вопрос 3: Аннотации Hibernate

**Вопрос:** Какие основные аннотации Hibernate используются на сущности Movie?

**Ваш ответ:**  
На сущности `Movie` используются такие основные аннотации:

1. `@Entity` — помечает класс как сущность JPA/Hibernate  
2. `@Table(name = "movies")` — задаёт имя таблицы в БД  
3. `@Id` — помечает поле первичным ключом  
4. `@GeneratedValue(strategy = GenerationType.IDENTITY)` — включает автогенерацию идентификатора  
5. `@Column(...)` — настраивает свойства конкретной колонки, например имя, `nullable`, длину

---

## Короткий вывод

- JDBC ближе к SQL и даёт больше ручного контроля.
- Hibernate даёт более высокий уровень абстракции и уменьшает количество boilerplate-кода.
- HQL работает с сущностями, а не с таблицами.
